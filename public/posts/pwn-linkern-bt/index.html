Pwn: Weaponizing LILYGO T-Embed for exploiting Bluetooth

2025-02-16yocvito
#pwn  #bluetooth  #linux 

It’s been some time we see uses of embedded devices for offensive security tasks (esp marauder, esp deauther, pwngotchi, …). This devices were highly capable in performing various attacks either directly on the RF PHY layer (eavesdropping, jamming, etc…), and/or upper protocols (WPA2 deauth frame, rolling codes, etc…). Most of the time, these devices were highly specialized.

With the apparition of Flipper Zero, a generalist pentesting device with extended capabilities compared to others, we saw a huge interest in developping custom Firmwares to improve the capabilities given by the hardware. For example, work have been done to create/extend applications to interact with BLE devices. For example the BadBLE rubber ducky module (first added in Xtreme Firmware by me) or the BLE spam application added by WillyJL.

It was mainly non harmful features which were added, but it shows that such devices (and it was already known with marauder) could definitely be used to attack everyday devices which interact with wireless protocols.

My main interest here is about the networking stacks which implement these protocols. This code often run with high privileges (kernel, system components) and thus, finding vulnerabilities in these codebases could potentially have high impact.
